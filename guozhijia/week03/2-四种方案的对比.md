# 项目方案
## 四种方案
  - 正则、TF-IDF + 机器学习、BERT 和 大模型api
## 方案对比
### 1. 正则
#### 优点
  - 速度极快
  - 规则配置好匹配成功无歧义，不会有意外输出
  - 资源消耗极低
#### 缺点
  - 扩展差，新加的功能需要人工添加
  - 只能处理编写好的规则
  - 无法理解同义词
  - 换个表达也无法识别
### 2. TF-IDF + 机器学习
#### 优点
  - 速度快
  - 有一定的理解能力
  - 资源消耗低
  - 模型小，可部署在终端
#### 缺点
  - 理解能力有限，同样无法理解同义词
  - 换个表达无法识别
  - 需要分词
### 3. BERT
#### 优点
  - 理解能力较强，具有较强的泛化能力
  - 识别率高
#### 缺点
  - 速度较慢
  - 模型较大，无法部署在端侧
  - 需要标注数据，数据质量极大影响模型效果
  - 资源消耗较大
### 4. 大模型api
#### 优点
  - 拓展性强
  - 只需写好提示词，无需训练
  - 理解能力强，具有很强的泛化能力
  - 无需部署在服务器
#### 缺点
  - 速度极慢
  - 成本高
  - 可能出现幻觉
  - 无法控制每次输出一模一样
